Detailed Description

The hardwareCanInterface class.

This is the Interface class that shall be used in order to implement a CAN Hardware Interface.

The CAN hardware interface is a class that interfaces a third party hardware device providing the Gantry device's the CAN communication connectivity.

The class shall implement the following methods:

  • hardwareCanInterface::hardwareInit(): initialize the hardware;
  • hardwareCanInterface::openRxFilter(): open a reception filter in the CAN device;
  • hardwareCanInterface::run(): activate the communication on the bus;
  • hardwareCanInterface::stop(): stop the communication on the bus;
  • hardwareCanInterface::driverTxRxData(): a method implementing the data tx/rx to the remote device

openRxFilter

This function shall open a device acceptance filter as part of the acceptance process in the CAN communication.

The address and mask usually are matched with the incoming CAN-ID 11bit for standard format or 29 bit for Extended format;

The acceptance rule is: (CAN-ID & mask) == address;

The implementation shall allow to use several time this method in order to open more than one acceptance filter.

Parameters
addressis the addreess filter
maskis the mask of the filter
